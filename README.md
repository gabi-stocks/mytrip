# מתכנן הטיולים

אפליקציית ווב פשוטה (בלי שלב build) לתכנון טיולים משותף: כמה טיולים, כמה משתתפים באותו קישור, מפת Google עם קטגוריות (לינה / אטרקציות / מסעדות / אחר), תאריכים אמיתיים לכל מקום, סדר וחישוב מרחקים, ייצוא לוורד, גיבוי/שחזור, ופח אשפה למחיקות בטעות.

מבנה הקבצים:
```
index.html            עמוד הבית — רשימת טיולים, יצירת טיול חדש, שחזור מגיבוי
trip.html              מרחב עבודה של טיול בודד — מפה + טבלה
css/styles.css         עיצוב
js/firebase-init.js    חיבור ל-Firebase (יש להחליף בפרטים שלך)
js/maps-loader.js      טעינת Google Maps (יש להחליף מפתח API)
js/trip-store.js       פעולות קריאה/כתיבה ל-Firestore, כולל פח אשפה (soft delete)
js/places-search.js    חיפוש מקומות והצעות (Google Places)
js/categories.js       שמות וצבעים של הקטגוריות (מקור אחד לכל האפליקציה)
js/date-utils.js       עיצוב תאריכים בעברית וחישוב טווח תאריכים
js/geo-utils.js        מרחק בין נקודות (ק"מ) וקישור ל-Google Maps
js/export-word.js      ייצוא הטיול למסמך Word מסודר לפי תאריכים
js/backup.js           גיבוי הטיול לקובץ JSON, ושחזור ממנו
js/app-list.js         לוגיקה של עמוד הבית
js/app-trip.js         לוגיקה של עמוד הטיול
```

אין שלב build (webpack/vite וכו') — כל הקבצים הם HTML/CSS/JS רגילים, בדיוק כמו הפרויקטים שלך ב-GitHub Pages.

## שלב 1 — הקמת פרויקט Firebase (בסיס הנתונים המשותף)

1. כנסו ל-https://console.firebase.google.com ולחצו "הוספת פרויקט".
2. בתוך הפרויקט: **Build → Firestore Database → צור מסד נתונים**. בחרו מצב "Production" (נגדיר כללי הרשאה בעצמנו למטה).
3. **Build → Authentication → Sign-in method** — הפעילו את השיטה **Anonymous**. זה מה שנותן לכל משתתף "זהות" בלי מסך הרשמה.
4. **Project settings (סמל גלגל השיניים) → General → Your apps → Web app (`</>`)**. תנו שם לאפליקציה, ותקבלו אובייקט `firebaseConfig` עם `apiKey`, `authDomain` וכו'.
5. פתחו את `js/firebase-init.js` והחליפו את הערכים ב-`firebaseConfig` באלה שקיבלתם.

### כללי הרשאה (Firestore Rules)
ב-**Firestore Database → Rules** הדביקו:
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /trips/{tripId} {
      allow read, write: if request.auth != null;
      match /places/{placeId} {
        allow read, write: if request.auth != null;
      }
    }
  }
}
```
**חשוב להבין את מודל האבטחה:** כל מי שמחובר (גם אנונימית) יכול לקרוא ולערוך כל טיול, כל עוד הוא יודע את מזהה הטיול (ה-ID שמופיע בקישור). זה בדיוק כמו שיתוף קובץ ב-Google Drive עם "כל מי שיש לו את הקישור" — נוח לשיתוף בין בני משפחה, אבל אל תשתפו קישור לטיול עם מידע רגיש, ותזכרו שמי שמקבל את הקישור יכול גם לערוך ולמחוק (אם כי מחיקה כרגע היא "רכה" — ראו סעיף פח האשפה למטה).

## שלב 2 — מפתח Google Maps

1. כנסו ל-https://console.cloud.google.com ובחרו פרויקט (או צרו חדש — אפשר להשתמש באותו פרויקט של Firebase, הם מבוססים על אותה תשתית Google Cloud).
2. **APIs & Services → Library** — הפעילו את **Maps JavaScript API** ואת **Places API**.
3. **APIs & Services → Credentials → Create credentials → API key**.
4. **מומלץ מאוד:** לחצו על המפתח שנוצר → **Application restrictions → HTTP referrers** → הוסיפו את הדומיין שבו תארחו את האתר (למשל `https://<username>.github.io/*`). כך המפתח לא ניתן לשימוש מאתרים אחרים.
5. שימו לב ל"חיוב" (Billing) — Google Maps Platform דורש חשבון חיוב מקושר, אך כולל קרדיט חינמי חודשי (מספיק בדרך כלל לשימוש אישי). כדאי להגדיר **תקציב התראה** (Budget alert) ב-Cloud Console כדי לא להיות מופתעים.
6. פתחו את `js/maps-loader.js` והחליפו את `YOUR_GOOGLE_MAPS_API_KEY` במפתח שלכם.

## שלב 3 — הרצה מקומית לבדיקה

הקבצים משתמשים ב-ES Modules (`type="module"`), שדפדפנים חוסמים כשפותחים ישירות מהדיסק (`file://`). צריך שרת מקומי פשוט:

```bash
cd travel-planner-app
python3 -m http.server 8080
```
ואז לפתוח בדפדפן: `http://localhost:8080`

## שלב 4 — פריסה ל-GitHub Pages

באותו תהליך שכבר מוכר לך מפרויקט הסורק:
1. `git init`, `git add .`, `git commit`, ודחיפה (`push`) לריפו חדש ב-GitHub.
2. **Settings → Pages** בריפו → מקור: הענף הראשי (`main`), תיקייה `/ (root)`.
3. האתר יעלה בכתובת `https://<username>.github.io/<repo-name>/`.
4. אל תשכחו לחזור לשלב 2.4 ולהוסיף את הכתובת הזו להגבלות של מפתח ה-API.

## מה חדש בסבב הזה

- **תאריכים אמיתיים במקום "יום 1/2/3":** כל מקום מקבל תאריך קלנדרי אמיתי (`date`). אם מגדירים לטיול תאריך התחלה וסיום בעת היצירה, לשוניות התאריכים נבנות אוטומטית לכל הטווח — גם לפני שהוספתם מקומות.
- **סינון לפי סוג לאורך כל הטיול:** בחירת הקטגוריה "לינה" + התאריך "כל התאריכים" מציגה את כל בתי המלון של הטיול, ממוינים כרונולוגית.
- **קישור ישיר ל-Google Maps:** שם כל מקום (בטבלה ובחלון המידע על המפה) הוא קישור שפותח את המיקום המדויק ב-Google Maps.
- **מספור ומרחקים:** לכל מקום אפשר להוסיף מספר סדר ביום (1, 2, 3...). האפליקציה מחשבת את המרחק הקווי-אווירי (ק"מ) בין מקום למקום הבא **באותו תאריך**, ומציגה אותו גם בטבלה וגם בחלון המידע על המפה. כשבוחרים תאריך ספציפי, גם מצוירת על המפה מסלול המחבר את הנקודות לפי הסדר. חשוב: זהו מרחק אווירי ולא מרחק נסיעה בפועל בכביש — שימושי כדי לאמוד עומס, לא כתחליף לניווט.
- **הערה אישית:** שדה נפרד מ"הסבר קצר", לכל מקום.
- **שיתוף עם המשפחה:** כפתור השיתוף פותח את תפריט השיתוף של הנייד (וואטסאפ/הודעות/מייל) כשזמין, ונופל לחזרה להעתקת קישור במחשב.
- **פח אשפה:** מחיקת מקום לא מוחקת אותה לצמיתות — היא עוברת ל"פח האשפה" (כפתור ייעודי), משם אפשר לשחזר או למחוק לצמיתות.
- **ייצוא לוורד:** כפתור "ייצוא לוורד" מוריד מסמך `.doc` מסודר לפי תאריכים, עם כל מקום, קטגוריה, הסבר, הערה אישית ומרחק למקום הבא.
- **גיבוי ושחזור:** כפתור "גיבוי (הורדת קובץ)" מוריד קובץ JSON עם כל תוכן הטיול. בעמוד הבית, "שחזור טיול מקובץ גיבוי" יוצר טיול חדש מתוך קובץ כזה — רשת ביטחון גם אם טיול שלם נמחק בטעות או שקרה משהו ב-Firebase.

## מודל הנתונים ב-Firestore

```
trips/{tripId}
  name: string
  startDate: string ("YYYY-MM-DD") | null   # אופציונלי — אם קיים, בונה את כל לשוניות התאריכים מראש
  endDate: string ("YYYY-MM-DD") | null
  createdAt: timestamp

trips/{tripId}/places/{placeId}
  name: string
  category: "lodging" | "attraction" | "restaurant" | "other"
  date: string ("YYYY-MM-DD") | null   # תאריך קלנדרי אמיתי; ריק = "ללא תאריך"
  order: number | null                 # מספור בתוך אותו תאריך, לצורך חישוב מרחקים
  notes: string                        # הסבר קצר (אופי הפעילות)
  personalNote: string                 # הערה אישית, נפרדת מההסבר
  lat, lng: number
  placeId: string | null               # מזהה Google Places, אם המקום אותר דרך חיפוש
  address: string
  sourceLink: string | null            # למשל קישור לדף המלון בבוקינג
  deleted: boolean                     # true = בפח האשפה
  deletedAt: timestamp | null
  createdAt: timestamp
```

## מה עדיין לא כלול (הרחבות אפשריות בהמשך)
- מחיקת טיול שלם או עריכת שם/תאריכי טיול קיים דרך הממשק
- הבחנה בין "צפייה בלבד" ל"עריכה" למשתתפים
- מרחק נסיעה אמיתי בכביש (כרגע מרחק אווירי בלבד — שדרוג אפשרי עם Google Distance Matrix API, שכרוך בעלות נוספת)
- ריקון אוטומטי של פח האשפה אחרי זמן מוגדר
